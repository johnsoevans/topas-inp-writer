# Symmetry Mode (Distortion Mode) Refinement

This topic is not covered in the TOPAS-Academic Technical Reference manual at all — it comes entirely from Dinnebier, Leineweber & Evans, *Rietveld Refinement: Practical Powder Diffraction Pattern Analysis using TOPAS* (De Gruyter, 2018), Chapter 8, "Symmetry Mode Refinements." It describes a way to build TOPAS `.inp`/`.str` structural models for a symmetry-lowering phase transition using group-theory-derived "distortion mode amplitudes" instead of ordinary `x`/`y`/`z` fractional coordinates. Unlike other reference files in this skill, this isn't a TOPAS-native keyword — it's an INP *coding pattern* generated by an external tool (ISODISTORT) that produces plain `prm`/`site` syntax, so anything here should be read as "how the generated code looks and what it means," not as documented TOPAS keywords.

## When this applies

Many materials undergo a *non-reconstructive* phase transition — a subtle structural change (octahedral tilting, an atom displacing off a high-symmetry site, order/disorder of a mixed site, or magnetic ordering) rather than a full bond-rearrangement. These transitions connect a high-symmetry **parent** structure (the **aristotype**, usually the high-temperature/low-pressure form) to a lower-symmetry **child** structure (a **hettotype**), related by a group–subgroup relationship. Describing the child structure conventionally (its own space group, its own choice of origin/cell, its own free `x y z` coordinates) is legitimate but can obscure the relationship to the parent and, for complex distortions, requires far more free parameters than the physics of the transition actually has degrees of freedom for.

Symmetry mode refinement instead describes the child structure as **the parent structure's coordinates plus the amplitudes of a small number of symmetry-motivated distortion patterns**. This isn't just bookkeeping: the free-energy change at a real phase transition is governed by the order parameters of the parent space group's irreducible representations (Landau theory), and mode amplitudes *are* those order parameters — so refining them is refining physically meaningful quantities, not arbitrary coordinates.

## Why this is worth using (advantages over `x y z`)

1. **Fewer parameters.** A child structure that needs 24 conventional `x y z` parameters may need only 5 mode amplitudes if the physical distortion only has 5 real degrees of freedom — this is the single biggest practical win, and can be the difference between a structure being determinable from powder data or not. (It doesn't always reduce parameter count — see the WO3 P4/ncc example below, where mode amplitudes and xyz parameters come out 1:1 — but it never costs more, and often costs much less.)
2. **A natural amplitude scale.** Mode amplitudes are normalized so that magnitude is roughly bounded (0 to ~4 in practice) and *directly reflects the size of the structural change* — a large amplitude means a large distortion, immediately interpretable, unlike raw fractional-coordinate shifts of very different physical scale.
3. **Systematic exploration of candidate symmetries.** Because different combinations of active/inactive modes correspond to different space groups, you can explore "what if this distortion were absent/present" by simply zeroing or refining particular mode amplitudes — see the genetic-algorithm and inclusion-run techniques below.
4. **Tooling does the hard group-theory work.** The ISODISTORT web suite (http://stokes.byu.edu/iso/isodistort.html) or the Bilbao Crystallographic Server (http://www.cryst.ehu.es/) derive the modes and generate ready-to-use TOPAS `.STR` text directly — you don't need to work out irreducible representations by hand to use this.

## Core group-theory vocabulary (only as much as needed to read generated code)

- A phase transition is associated with a **k-point** (a wavevector in reciprocal space) — this is what determines the superlattice reflections seen after the transition (e.g. a transition at k=(½,0,0) doubles the a-axis and produces (h/2,k,l) superlattice peaks).
- For a given parent space group, a chosen k-point defines a set of **irreducible representations (irreps)** — essentially, the possible "shapes" a symmetry-breaking distortion can take.
- Selecting one irrep (the **primary irrep**) determines a candidate **isotropy subgroup** (the distortion's resulting symmetry) — but activating it can also activate **secondary irreps**, which may or may not have significant amplitude in the real structure.
- Within one irrep, different **order parameter directions (OPDs)** correspond to different remaining symmetries: the fully general OPD gives the lowest possible symmetry (the **kernel**); more constrained OPDs (e.g. requiring two components equal) give progressively higher intermediate symmetries. The free components of an OPD are called **branches**.
- A **distortion mode** is a vector component along one of the irrep's basis directions; its refined value is the **mode amplitude**.
- A useful chemist's-intuition analogy from the book: H2O's normal vibrational modes (symmetric stretch a1, antisymmetric stretch b2, bend a1) are exactly this kind of basis — describing a statically distorted molecule as the undistorted parent geometry plus a frozen amount of one or more of these modes is the same idea as a symmetry-mode structural refinement, just applied to a crystal instead of a molecule.

## The TOPAS `.STR` code pattern (ISODISTORT-generated)

ISODISTORT (having been given the parent structure and the target child space group/cell/origin) writes out a `.STR` file with four `'{{{ ... '}}}` folded blocks (foldable in the jEdit-based TOPAS editor so they don't need to be read/edited by hand):

```
'{{{mode definitions
   prm    a7  1.12853 min -4.00 max 4.00 'Pm-3m[1/2,1/2,1/2]R4+(a,0,b)[O:d]Eu(a)
   prm    a8  1.11797 min -4.00 max 4.00 'Pm-3m[1/2,1/2,1/2]R4+(a,0,b)[O:d]Eu(b)
'}}}
'{{{mode-amplitude to delta transformation
   prm O_1_dx = +0.03125*a8 - 0.03125*a10 + ...;: 0.03494
'}}}
'{{{distorted parameters
   prm O_1_x = 1/4 + O_1_dx;: 0.28494
'}}}
'{{{mode-dependent sites
   site O_1 x = O_1_x;:0.28494 y = O_1_y;:0.75000 z = O_1_z;:0.78527 occ O 1 beq 0
'}}}
```

- **Mode definitions**: one `prm` per refinable mode amplitude, named arbitrarily (`a7`, `a8`, ...). The trailing comment is the ISODISTORT mode label (see below) — informational only, not parsed by TOPAS.
- **Mode-amplitude to delta transformation**: linear equations converting mode amplitudes into fractional-coordinate *shifts* (`O_1_dx` etc.) — this is where the group-theory content actually lives, as fixed numeric coefficients on each mode.
- **Distorted parameters**: adds each shift to the parent's ideal (usually simple, e.g. `1/4`) coordinate to get the actual refined coordinate.
- **Mode-dependent sites**: ordinary `site` definitions referencing the distorted-parameter equations — from here down it's a completely normal TOPAS structure.

**Reading an ISODISTORT mode label**, e.g. `Pm-3m[1/2,1/2,1/2]R4+(a,0,b)[O:d]Eu(a)`: parent space group (`Pm-3m`) → k-point (`[1/2,1/2,1/2]`, the R-point) → irrep (`R4+`, using Miller & Love 1967 notation) → OPD (`(a,0,b)`) → the parent Wyckoff site being distorted (`[O:d]`) → the site's local point-group irrep inducing the distortion (`Eu`) → which branch of the OPD (`(a)` vs `(b)`).

## Worked example: WO3 (M-point and R-point distortions)

WO3's hypothetical cubic parent (space group Pm-3m, W on 1a, O on 3d, no free coordinates) undergoes a real sequence of cooling transitions via:
- An **M-point** irrep (M3−) describing W displacing off-centre within its WO6 octahedron (a 2nd-order Jahn–Teller distortion) — 7 possible M-point irrep choices exist, only one (M3−) matches the real WO3 behavior.
- **R-point** irreps (R4+) describing coupled octahedral tilts.

For the M3− irrep alone, six different OPD choices are possible, each giving a different child space group/cell/origin (from the fully general kernel S1(a;b;c) → I222, down to more constrained cases like P1(a;0;0) → P4/nmm, the actual ~1173 K WO3 phase). This is a concrete illustration of how choosing a *branch* of an OPD is choosing a specific candidate symmetry to test against the data — and why exploring several OPDs systematically (rather than guessing one xyz model) is valuable.

A real refined example (P4/ncc WO3 at 850°C) uses just 3 mode amplitudes (2 for the M-point W/O off-centering, 1 for the R-point tilt) — in this particular case that's the *same* parameter count as a conventional xyz description (3 free coordinates), so the parameter-reduction advantage isn't universal, but the systematic-symmetry-exploration advantage still applies: refining with all mode amplitudes fixed at zero reproduces the P4/mmm subgroup fit exactly, and fixing lattice parameters to their cubic ratios on top of that reproduces the parent Pm-3m fit exactly — the same INP file family can represent parent, an intermediate subgroup, and the true child model just by switching which mode amplitudes are allowed to refine.

At room temperature (P21/n WO3), the parameter-reduction advantage is real and large: a conventional 24-parameter xyz refinement gives Rwp=9.0%, while a 5-mode-amplitude symmetry-mode refinement gives an almost identical Rwp=9.2% — the 5 modes (tungsten X5− and M3− off-centre displacements, 2× oxygen R4+ and 1× oxygen M3+ octahedral rotations) capture essentially all of the physically real distortion.

## Determining which modes matter when the symmetry is *not* already known

If you don't already know the true child space group, three practical approaches (all discussed in the book with reference to a WO3 case with 96 possible mode amplitudes in a hypothetical 2×2×2 P1 description):

1. **Genetic algorithm (GA).** Represent a candidate model as a bit-string of 96 "on/off" flags (which modes are allowed to refine). Refine each candidate, rank by Rwp (with a small penalty added per active mode to favor simpler models), then apply selection/crossover/mutation across a population of candidate bit-strings across generations. For WO3 this converges (within minutes on a desktop) to a 7-mode model; feeding that result to the FINDSYM algorithm (http://stokes.byu.edu/iso/findsym.php) identifies the true symmetry as P21/n.
2. **Inclusion runs.** Conceptually simpler: refine each of the 96 modes individually (single-parameter fits), keep whichever gives the lowest Rwp, then refine it jointly with each of the remaining 95 in turn (2-parameter fits), and repeat — building up an ordered sequence of modes by marginal Rwp improvement. Plotting Rwp vs. number-of-modes-included typically shows a sharp elbow (for WO3, at 7 modes) after which further modes barely help; the same 7 modes as the GA approach, and the process is fully reversible (starting from all 96 modes on and removing the least-useful one at a time gives the same curve).
3. **Exhaustive subgroup-tree search.** ISODISTORT can enumerate *every* possible intermediate subgroup between a parent and a low-symmetry base structure with enough degrees of freedom to fully explain the data (for the WO3 Pm-3m → 2×2×2 P1 case, this is 1427 distinct candidate structures) — each can be tested against the data to map the whole Rwp surface and pick the simplest structure that fits well. Described further in Lewis et al. (2016).

Both the GA and inclusion-run methods, applied to the same real WO3 data, converge on the same 7 modes and the same P21/n symmetry — a useful cross-check when trying this on a new/unknown material.

## Worked example: Mg(H2O)6RbBr3 (a full ISODISTORT → TOPAS.STR cookbook)

The book walks through a complete practical recipe:
1. Upload the ordered high-symmetry parent structure (as a CIF) to the ISODISTORT server.
2. Use ISODISTORT "method 3" ("search over arbitrary k points for specified space group and lattice") to find the transformation from the parent to the known low-temperature child space group (here C2/c), specifying the child unit-cell basis as an integer transformation matrix on the parent cell vectors.
3. Pick the matching basis/origin choice from the list ISODISTORT offers, then export a `TOPAS.STR` file directly — this text drops straight into an INP file's structural block in place of ordinary `site` coordinate definitions.
4. After refinement, ISODISTORT's "method 4" (mode decomposition of a distorted structure from a CIF) can compare parent and refined child structures directly, identify which mode amplitudes are largest (i.e. structurally most significant), and animate the parent→child transition in its own viewer.

In this example, 13 possible mode amplitudes correspond exactly to the 13 free fractional coordinates of the conventional description — again showing the parameter count doesn't always shrink, but the physical interpretability (which 2 of the 13 modes actually dominate the real distortion) is the payoff.

## A real limitation: modes vs. true rigid-body rotation

A subtlety worth knowing before over-interpreting refined mode amplitudes as literal rigid-body rotations: an ISODISTORT displacive mode describing an "octahedral tilt" actually just displaces the ligand atoms — it also slightly *distorts* the polyhedron (e.g. a small expansion) rather than rotating it as a perfectly rigid unit, since it doesn't enforce constant bond lengths. Properly describing a fully rigid rotation requires a distinct, more complex technique — **rotational rigid-body modes** (Müller et al., 2014), where a rotation axis is defined as a vector in space with the vector's length encoding rotation magnitude. This is implemented in TOPAS but is a materially different (and more involved) formalism than the basic displacive-mode pattern described above; consult Müller et al. (2014) directly if a refinement genuinely needs rigid rotation rather than an approximate displacive mode.

## Extending beyond atomic displacements

The same order-parameter language extends to other physical quantities with different tensor character: site **occupancies** (order/disorder transitions, a scalar order parameter), **magnetic ordering** (an axial-vector/pseudo-vector order parameter — see `references/12-magnetic-structure-refinement.md` and the book's Chapter 9, which uses this same symmetry-mode machinery for magnetic Shubnikov-group determination), and rigid-body **rotational** order parameters (axial vectors, per the Müller et al. rotational-mode formalism above).
